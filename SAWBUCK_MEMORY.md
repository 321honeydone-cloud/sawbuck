# Sawbuck Memory

This file is Sawbuck's long-term brain. The LESSONS section gets injected into
the estimator prompt on every single chat. The JOB LOG below is the raw record:
every estimate that completes, gets sent, gets invoiced, or gets DELETED lands
here first, plus every price correction Manny makes. Every 10 events the log is
automatically distilled into LESSONS by the fast model.

You can edit LESSONS by hand any time. Sawbuck treats what's written there as
shop truth. Deleting an estimate in the app does NOT remove its entry here.

## Lessons

- QUOTE FORMAT (permanent, applies to every estimate): always present TWO BUILDS in one quote, never a single worst-case number. (1) Smooth Scenario, the honest expected price if the work goes as planned, is the base scope and the number to lead with. (2) Max Cost Guarantee, the absolute ceiling, equals the Smooth Scenario plus itemized if-needed allowances, each tied to one specific complication and only charged if that exact issue is found. Build the base work in normal trade groups, then a group named "Complications Cap (only if needed)" with one line item per allowance whose name states the trigger. Lead with the Smooth Scenario price, then state the Max Cost Guarantee as the most they will ever pay, and frame allowances as client protection, not padding. The Max Cost Guarantee (everything included) must stay under $2,500 per job. Never split a job to slip under that cap.
